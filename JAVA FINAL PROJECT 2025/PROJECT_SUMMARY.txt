==============================================
CAMPUS PLACEMENT MANAGEMENT SYSTEM
Final Year Project - Project Summary
==============================================

PROJECT OVERVIEW:
================
This project implements a comprehensive Campus Placement Management System 
that facilitates communication between students, recruiters, and placement officers.

TECHNOLOGY STACK:
================
Backend:
- Java 17
- Spring Boot 3.1.0
- Spring Data JPA
- MySQL Database
- Maven Build Tool

Frontend:
- HTML5
- CSS3
- JavaScript (Vanilla)
- Responsive Design with Beautiful UI

Features Implemented:
====================

1. ROLE-BASED AUTHENTICATION
   - Student Login/Registration
   - Recruiter Login/Registration  
   - Placement Officer Login/Registration
   - Secure session management

2. STUDENT DASHBOARD
   - Browse available job postings
   - Apply for jobs
   - Track application status
   - View interview details
   - Real-time updates

3. RECRUITER DASHBOARD
   - Post new job openings
   - Set eligibility criteria (CGPA, skills, etc.)
   - View student applications
   - Manage job postings
   - Track application status

4. PLACEMENT OFFICER DASHBOARD
   - View all applications
   - Shortlist candidates
   - Reject applications
   - Statistical dashboard
   - Interview management

5. API ENDPOINTS
   - RESTful API design
   - Complete CRUD operations
   - JSON-based communication
   - CORS enabled for frontend

DATABASE DESIGN:
================
Tables Created:
1. users - Stores all user information
2. jobs - Stores job postings
3. applications - Stores student job applications
4. interviews - Stores interview details

HOW TO RUN:
==========
1. Install Java 17, Maven, MySQL
2. Create database: placement_db
3. Update database credentials in application.properties
4. Run: mvn spring-boot:run
5. Access: http://localhost:8080

API ENDPOINTS:
==============
Authentication:
- POST /api/auth/register
- POST /api/auth/login

Jobs:
- GET /api/jobs
- GET /api/jobs/{id}
- POST /api/jobs
- PUT /api/jobs/{id}/status
- DELETE /api/jobs/{id}

Applications:
- GET /api/applications
- POST /api/applications
- GET /api/applications/student/{id}
- GET /api/applications/job/{id}
- PUT /api/applications/{id}/status

Interviews:
- GET /api/interviews
- POST /api/interviews
- PUT /api/interviews/{id}

Users:
- GET /api/users
- GET /api/users/{id}
