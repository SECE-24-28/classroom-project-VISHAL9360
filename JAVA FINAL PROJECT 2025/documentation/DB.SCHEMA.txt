╔══════════════════════════════════════════════╗
║          ATTRIBUTE DESCRIPTIONS              ║
╚══════════════════════════════════════════════╝

USER TABLE:
-----------
- id: Primary key
- username: Unique username for login
- password: Encrypted password
- email: User email address
- full_name: User's full name
- role: STUDENT | RECRUITER | PLACEMENT_OFFICER
- phone: Contact number
- department: For students only (CS, IT, ECE, etc.)
- cgpa: For students only (0-100 scale)
- resume_path: Path to uploaded resume file
- company_name: For recruiters only
- designation: For recruiters only

JOB TABLE:
----------
- id: Primary key
- title: Job position title
- description: Detailed job description
- company_name: Company offering the job
- location: Job location
- salary: Salary range/package
- job_type: Full-time | Internship | Part-time
- min_cgpa: Minimum CGPA requirement
- required_skills: Comma-separated skills
- eligibility: Eligibility criteria text
- posted_by: Foreign key to users table
- application_deadline: Last date to apply
- status: ACTIVE | CLOSED

APPLICATION TABLE:
------------------
- id: Primary key
- student_id: Foreign key to users (student)
- job_id: Foreign key to jobs
- status: PENDING | SHORTLISTED | SELECTED | REJECTED
- resume_path: Resume file path
- cover_letter: Cover letter text
- applied_at: Application submission date
- updated_at: Last update timestamp

INTERVIEW TABLE:
----------------
- id: Primary key
- application_id: Foreign key to applications
- round: Interview round (Technical, HR, etc.)
- scheduled_at: Interview date and time
- location: Interview location
- status: SCHEDULED | COMPLETED | CANCELLED
- result: PASSED | FAILED | PENDING
- feedback: Interview feedback notes
- created_at: Record creation date

╔══════════════════════════════════════════════╗
║          CONSTRAINTS                         ║

Primary Keys:
- All tables have 'id' as primary key (auto-increment)

Foreign Keys:
- applications.student_id REFERENCES users.id
- applications.job_id REFERENCES jobs.id
- jobs.posted_by REFERENCES users.id
- interviews.application_id REFERENCES applications.id

Indexes:
- users.username (unique)
- users.email (unique)
- applications (student_id, job_id) - composite
- jobs.posted_by
- interviews.application_id
