==============================================
CAMPUS PLACEMENT MANAGEMENT SYSTEM
Final Year Project - Project Summary
==============================================

PROJECT OVERVIEW:
================
This project implements a comprehensive Campus Placement Management System 
that facilitates communication between students, recruiters, and placement officers.

TECHNOLOGY STACK:
================
Backend:
- Java 17
- Spring Boot 3.1.0
- Spring Data JPA
- MySQL Database
- Maven Build Tool

Frontend:
- HTML5
- CSS3
- JavaScript (Vanilla)
- Responsive Design with Beautiful UI

Features Implemented:
====================

1. ROLE-BASED AUTHENTICATION
   - Student Login/Registration
   - Recruiter Login/Registration  
   - Placement Officer Login/Registration
   - Secure session management

2. STUDENT DASHBOARD
   - Browse available job postings
   - Apply for jobs
   - Track application status
   - View interview details
   - Real-time updates

3. RECRUITER DASHBOARD
   - Post new job openings
   - Set eligibility criteria (CGPA, skills, etc.)
   - View student applications
   - Manage job postings
   - Track application status

4. PLACEMENT OFFICER DASHBOARD
   - View all applications
   - Shortlist candidates
   - Reject applications
   - Statistical dashboard
   - Interview management

5. API ENDPOINTS
   - RESTful API design
   - Complete CRUD operations
   - JSON-based communication
   - CORS enabled for frontend

DATABASE DESIGN:
================
Tables Created:
1. users - Stores all user information
2. jobs - Stores job postings
3. applications - Stores student job applications
4. interviews - Stores interview details

HOW TO RUN:
==========
1. Install Java 17, Maven, MySQL
2. Create database: placement_db
3. Update database credentials in application.properties
4. Run: mvn spring-boot:run
5. Access: http://localhost:8080

API ENDPOINTS:
==============

- GET /api/applications
- POST /api/applications

Interviews:
- GET /api/interviews
- POST /api/interviews
- PUT /api/interviews/{id}

Users:
- GET /api/users
- GET /api/users/{id}

╔══════════════════════════════════════════════╗
║          ATTRIBUTE DESCRIPTIONS              ║
╚══════════════════════════════════════════════╝

USER TABLE:
-----------
- id: Primary key
- username: Unique username for login
- password: Encrypted password
- email: User email address
- full_name: User's full name
- role: STUDENT | RECRUITER | PLACEMENT_OFFICER
- phone: Contact number
- department: For students only (CS, IT, ECE, etc.)
- cgpa: For students only (0-100 scale)
- resume_path: Path to uploaded resume file
- company_name: For recruiters only
- designation: For recruiters only

JOB TABLE:
----------
- id: Primary key
- title: Job position title
- description: Detailed job description
- company_name: Company offering the job
- location: Job location
- salary: Salary range/package
- job_type: Full-time | Internship | Part-time
- min_cgpa: Minimum CGPA requirement
- required_skills: Comma-separated skills
- eligibility: Eligibility criteria text
- posted_by: Foreign key to users table
- application_deadline: Last date to apply
- status: ACTIVE | CLOSED

APPLICATION TABLE:
------------------
- id: Primary key
- student_id: Foreign key to users (student)
- job_id: Foreign key to jobs
- status: PENDING | SHORTLISTED | SELECTED | REJECTED
- resume_path: Resume file path
- cover_letter: Cover letter text
- applied_at: Application submission date
- updated_at: Last update timestamp

INTERVIEW TABLE:
----------------
- id: Primary key
- application_id: Foreign key to applications
- round: Interview round (Technical, HR, etc.)
- scheduled_at: Interview date and time
- location: Interview location
- status: SCHEDULED | COMPLETED | CANCELLED
- result: PASSED | FAILED | PENDING
- feedback: Interview feedback notes
- created_at: Record creation date

╔══════════════════════════════════════════════╗
║          CONSTRAINTS                         ║
╚══════════════════════════════════════════════╝

Primary Keys:
- All tables have 'id' as primary key (auto-increment)

Foreign Keys:
- applications.student_id REFERENCES users.id
- applications.job_id REFERENCES jobs.id
- jobs.posted_by REFERENCES users.id
- interviews.application_id REFERENCES applications.id

Indexes:
- users.username (unique)
- users.email (unique)
- applications (student_id, job_id) - composite
- jobs.posted_by
- interviews.application_id
